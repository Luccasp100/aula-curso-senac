Aula 9: Formulários e Validação de Dados

Capturando e validando a entrada do usuário de forma interativa.

O Evento Chave: 'submit'

Para formulários, não ouvimos o 'click' no botão. Ouvimos o evento 'submit' no próprio elemento <form>.

const meuFormulario = document.getElementById('meu-form');

meuFormulario.addEventListener('submit', function(event) {
    // O código aqui dentro roda quando o form é enviado.
});

Isso garante que a validação ocorra tanto ao clicar no botão quanto ao pressionar 'Enter' em um campo.

Assumindo o Controle: event.preventDefault()

Por padrão, o navegador recarrega a página ao submeter um formulário. Para que nosso JavaScript possa fazer a validação, precisamos impedir essa ação.

meuFormulario.addEventListener('submit', function(event) {
    event.preventDefault(); // A primeira linha, SEMPRE!

    console.log("Formulário enviado, mas a página não recarregou!");
});

Lendo os Dados: A Propriedade .value

Dentro do nosso listener, podemos acessar o que o usuário digitou em um campo de input através da propriedade .value.

const inputNome = document.getElementById('campo-nome');

meuFormulario.addEventListener('submit', function(event) {
    event.preventDefault();
    const nomeDigitado = inputNome.value;
    console.log(`O nome digitado foi: ${nomeDigitado}`);
});

Feedback Visual: Manipulando Classes CSS

A melhor forma de indicar erros é adicionando ou removendo classes CSS nos elementos. Isso separa a lógica (JS) da aparência (CSS).

CSS: 
.input-error { border: 2px solid red; }
.msg-error { color: red; display: block; }
JavaScript: 
if (nomeDigitado === "") {
    inputNome.classList.add('input-error');
    mensagemDeErro.textContent = "Campo obrigatório!";
} else {
    inputNome.classList.remove('input-error');
}
